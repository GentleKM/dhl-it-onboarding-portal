# AI 에이전트 정책

## 역할 분담

- **Claude**: 개발자 역할. 코드 작성 및 구현 담당.
- **Codex**: 기획자/검토자 역할. 코드 리뷰, 보안 검토, UX 검토 담당.

## 무한 루프 방지

- 코딩 → 실패 → 코딩 반복이 5회 넘어가면 즉시 멈추고 사용자에게 보고

## 작업 순서 원칙

- 큰 작업은 5~10개 Phase로 분리해 순차 진행
- 각 Phase 완료 후 status.json에 결과 기록

## 브랜치 보호

- main 브랜치 강제 푸시(git push -f) 절대 금지
- 검증되지 않은 코드는 main에 직접 병합 금지

## 동일한 실수 반복 방지

- 동일한 오류를 3회 반복하면 해당 오류 패턴과 해결책을 rules/ 파일에 즉시 명문화

## 보안 원칙

- API 키 등 민감 정보는 프롬프트에 직접 입력하지 않음
- 환경 변수·Secret 관리 도구로 격리
