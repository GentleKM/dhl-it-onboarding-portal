# 테스팅 규칙

## TDD 원칙

- 기능 구현 전 테스트 코드 먼저 작성
- 구현이 없는 상태에서 테스트가 실패해야 함 (Red → Green → Refactor)

## 테스트 범위

- AI 추천 로직: 4개 솔루션이 올바르게 분류되는지 시나리오 테스트
- DB 저장/조회: Supabase 연동 통합 테스트
- API Route: 입력 검증 및 에러 처리 테스트

## 검증되지 않은 코드

- 검증되지 않은 코드는 main 브랜치에 직접 병합 금지
- Git Worktree 등 별도 환경에서 테스트 후 병합

## 보고 방식

- 성공 중심이 아닌 '실패' 중심으로 보고
- 피드백: "기대한 결과 vs 실제 결과"의 격차로 구체적으로 설명
